## Repository Settings

Configure your repository to match your project's needs and team workflow.

<Steps>
  <Step title="Access Settings" icon="settings">
    Navigate to your repository's Settings tab on GitHub.com.
  </Step>
  <Step title="General Settings" icon="file-text">
    Configure basic repository information like description and website.
  </Step>
  <Step title="Branch Protection" icon="shield">
    Set up branch protection rules to prevent accidental changes.
    
    ```yaml
    # Example branch protection
    Require pull request reviews before merging
    Require status checks to pass
    Include administrators
    ```
  </Step>
  <Step title="Collaborators" icon="users">
    Manage team access and permissions.
  </Step>
</Steps>

## Git Configuration

Optimize your local Git setup for better repository management.

<Tabs>
  <Tab title="Global Config" icon="settings">
    <Expandable title="User Configuration" default-open="true">
      Set global Git configuration for consistent behavior across all repositories.
      
      ```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
git config --global core.editor "code --wait"
      ```
    </Expandable>
  </Tab>
  <Tab title="Repository Config" icon="folder">
    <Expandable title="Local Repository Settings" default-open="true">
      Configure settings specific to individual repositories.
      
      ```bash
git config user.name "Different Name"  # Override global
      ```
    </Expandable>
  </Tab>
</Tabs>

## GitHub Actions Configuration

Set up automated workflows to streamline your development process.

<Callout kind="tip">Use GitHub Actions marketplace to find pre-built actions for common tasks.</Callout>

<CodeGroup tabs="Testing,Deployment">
  ```yaml
  # .github/workflows/test.yml
  name: Test
  on: [push, pull_request]
  jobs:
    test:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v3
        - uses: actions/setup-node@v3
          with:
            node-version: '18'
        - run: npm ci
        - run: npm test
  ```
  ```yaml
  # .github/workflows/deploy.yml
  name: Deploy
  on:
    push:
      branches: [main]
  jobs:
    deploy:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v3
        - run: npm run build
        - run: npm run deploy
  ```
</CodeGroup>
